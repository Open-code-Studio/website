关于 CloudFlow

了解 CloudFlow 的诞生背景、架构设计与发展规划

/ 诞生背景

为什么打造 CloudFlow

服务器监控是运维工作中的核心环节。然而，现有的 Linux 监控工具往往面临两个极端：要么功能强大但配置复杂、学习曲线陡峭；要么界面简陋、可视化能力不足且缺乏跨平台支持。

CloudFlow 的目标是填补这一空白。我们希望提供一款既具备企业级监控能力，又拥有现代、直观用户体验的工具。基于 Dart/Flutter 构建，CloudFlow 在保持高性能的同时，实现了桌面端与 Web 端的统一体验。

/ 系统架构

架构设计

数据采集层（Dart）

基于 Dart 的高性能采集引擎，通过 systemd 接口和 /proc 文件系统获取系统指标，支持插件式扩展采集项。

存储层（InfluxDB）

使用 InfluxDB 时序数据库存储监控数据，支持高效的数据写入与查询，自动数据保留策略。

通信层（gRPC + WebSocket）

采集端与控制端通过 gRPC 通信保障数据可靠传输，前端通过 WebSocket 获取实时数据流。

展示层（Flutter）

基于 Flutter 构建统一 UI，桌面端与 Web 端共享同一套代码，确保跨平台一致体验。

/ 开发路线

版本里程碑

v0.1 Alpha — 基础监控面板，CPU/内存/磁盘实时数据展示

v0.2 Beta — 自定义仪表盘编辑器，支持拖拽布局

v0.3 — 告警引擎上线，支持阈值告警和通知推送

v0.4 — 多服务器管理，支持采集端自动发现

v0.5 — 插件式数据采集，社区扩展能力开放
