关于 JMCL

了解 JMCL 的诞生背景、架构设计与开发历程

/ 诞生背景

为什么打造 JMCL

Minecraft 启动器是每位玩家进入游戏的必经门户。然而，市面上的启动器往往存在体验不一致、跨平台支持不完善、Mod 管理繁琐等问题。官方启动器功能基础，第三方启动器又可能捆绑不必要的功能或广告。

JMCL（Java Minecraft Launcher）的目标是提供一款真正轻量、开源、跨平台的 Minecraft 启动器。基于 Java 构建，JMCL 天然具备跨平台能力，同时保持极小的体积和干净的界面。我们希望让玩家专注于 Minecraft 本身，而不是被启动器困扰。

/ 架构设计

系统架构

UI 层（JavaFX）

使用 JavaFX 构建现代化图形界面，支持 CSS 样式定制，提供流畅的动画和响应式布局。

版本管理层

自动从 Mojang 元数据服务获取版本列表，支持版本缓存、增量更新和自定义版本目录。

Mod 加载引擎

内置 Forge 和 Fabric 安装器，自动处理加载器版本匹配和 Mod 依赖关系，简化 Mod 管理。

诊断与日志

提供启动日志实时查看、Java 环境检测、常见问题诊断，帮助用户快速定位和解决问题。

/ 开发路线

版本里程碑

v0.1 Alpha — 基础启动器核心，支持版本管理和一键启动

v0.2 Beta — Forge/Fabric 加载器支持，Mod 管理功能

v0.3 — 账号系统上线，支持微软/Mojang 登录

v0.4 — 资源包与光影包管理，游戏配置可视化编辑

v0.5 — 插件系统开放，支持社区扩展启动器功能
